# Cookbook Collection

## Описание проекта
Cookbook Collection — это веб-приложение для управления кулинарными рецептами. 
Пользователи могут сохранять рецепты в различных категориях, редактировать их. 
Приложение поддерживает авторизацию и сохранение уникальных данных для каждого пользователя, включая функциональность загрузки аватаров и изображений для рецептов.

## Основные технологии
### Frontend
- **React**: построение динамических компонентов.
- **React Router**: организация маршрутизации между страницами (книги, рецепты, страница пользователя).
- **SCSS**: стилизация компонентов.
- **Axios**: работа с внешними API через HTTP-запросы.
- **AuthContext**: управление состоянием авторизации (имя, email, аватар, книга и т.д.).

### Backend
- **Express.js**: серверная часть на Node.js.
- **MongoDB**: база данных для хранения пользователей и рецептов.
- **Mongoose**: ORM для работы с MongoDB.
- **Multer**: для обработки загрузки файлов (изображений).
- **JWT (JSON Web Tokens)**: для аутентификации.
- **CORS**: для управления доступом к API.
- **Body-parser**: для парсинга тела запросов.

## Основные страницы
1. **Главная страница (`Home`)**:
   - Отображает список кулинарных книг.
   - Пользователь может выбрать книгу для просмотра рецептов.
   - Поддержка отображения имени пользователя и аватара.

2. **Страница книги (`Book`)**:
   - Отображает рецепты, связанные с выбранной книгой.
   - Включает кнопки возврата в меню и навигации на главную страницу.

3. **Страница рецепта (`Recipe`)**:
   - Отображает полный рецепт с ингредиентами и инструкцией.
   - Пользователь может редактировать или удалять рецепт при необходимости.

4. **Страница пользователя (`UserPage`)**:
   - Позволяет пользователю просматривать и редактировать свои данные (имя, email, аватар).
   - Пользователь может изменить аватар с возможностью предварительного просмотра перед сохранением.
   - Реализована возможность удаления учётной записи.

5. **Страница поиска (`SearchResults`)**:
   - Отображает результаты поиска по запросу, введённому пользователем.
   - Использует `iframe` для встраивания результатов поиска Google по введённому запросу.

## Функции
- **Авторизация**:
   - Реализована через JWT токены, которые хранятся в `localStorage`.
   - Данные пользователя автоматически подгружаются при входе в систему.

- **Редактирование пользователя**:
   - Пользователь может изменить своё имя, email и аватар.
   - Возможность предварительного просмотра аватара перед загрузкой.

- **Управление рецептами**:
   - Пользователи могут добавлять, редактировать и удалять свои рецепты.
   - Поддержка загрузки изображений для каждого рецепта.

- **Поиск**:
   - Реализован поиск через Google по ключевым словам, введённым пользователем.
